### Integration Issues (Tech Debt)

**1. Budget email notifications never triggered**
- `checkAndNotifyBudgets()` and `sendAdminDigest()` in `src/lib/server/budget/notifications.ts` are fully implemented but never called
- Gateway endpoints ignore `softLimitHit` flag from `checkBudget()` result
- In-app `BudgetBanner` works correctly (separate cascade query in layout)
- Deferred by design: STATE.md notes "Admin digest scheduling deferred to Phase 5 — function ready, needs cron trigger"
- **Impact:** Members don't receive email alerts; admins don't get daily digest. In-app banner is the only notification channel.

**2. LiteLLM not bundled in docker-compose**
- `docker-compose.yml` defines app + postgres + redis (3 services)
- `proxy.ts` forwards to `LITELLM_API_URL` (defaults to `http://localhost:4000`)
- The self-host guide documents `LITELLM_API_URL` as a required env var
- **Impact:** `docker compose up -d` alone won't produce a fully working gateway — LiteLLM must be started separately
