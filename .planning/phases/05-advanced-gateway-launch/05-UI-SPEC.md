# Phase 5 — UI Design Contract

> Visual and interaction contract for the landing page, smart routing settings, cache TTL configuration, API key smart routing toggle, and integration docs page. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none |
| Preset | not applicable |
| Component library | none (hand-rolled Svelte components with Tailwind) |
| Icon library | Inline SVG (Lucide-style, 16px default, stroke-width 2) |
| Font | system-ui, -apple-system, BlinkMacSystemFont, 'Segoe UI', sans-serif |

Carried forward from Phase 4 — no changes.

---

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding |
| sm | 8px | Compact element spacing, table cell padding vertical, badge padding |
| md | 16px | Default element spacing, card padding, table cell padding horizontal |
| lg | 24px | Section padding, card internal sections, panel internal padding |
| xl | 32px | Layout gaps, page section breaks |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level spacing, landing page section vertical padding |
| 4xl | 80px | Landing page hero vertical padding |
| 5xl | 96px | Landing page top-level vertical breathing room |

Exceptions: Landing page hero section uses 96px top padding and 64px bottom padding. Features grid uses 32px gap between cards. Cost comparison table uses 16px cell padding. Integration docs page uses 24px between steps.

#### SmartRoutingSettings Component

Props:
- `cheapModel: string | null` — current cheap model value
- `expensiveModel: string | null` — current expensive model value
- `orgSlug: string`

Form action: `POST /org/{orgSlug}/settings?/saveRouting`

Inputs:
- `cheapModel`: text input, `name="cheapModel"`, placeholder "e.g., gpt-4o-mini"
- `expensiveModel`: text input, `name="expensiveModel"`, placeholder "e.g., gpt-4o"

States:

| State | Behavior |
|-------|----------|
| Default | Inputs show current values or empty with placeholder |
| Saving | Button shows "Saving..." with spinner, inputs disabled (`opacity-50`) |
| Success | Green "Settings saved" text appears next to button, fades after 3s |
| Error | Toast: "Failed to save routing settings. Try again." |
| Empty | Both fields empty — smart routing will use the exact model requested (effectively disabled at org level) |

#### CacheTtlSetting Component

Props:
- `cacheTtlSeconds: number` — current TTL value (default 3600)
- `orgSlug: string`

Form action: `POST /org/{orgSlug}/settings?/saveCacheTtl`

Input:
- `cacheTtlSeconds`: number input, `name="cacheTtlSeconds"`, `min="60"`, `max="86400"`

States:

| State | Behavior |
|-------|----------|
| Default | Input shows current value (3600) |
| Saving | Button shows "Saving..." with spinner, input disabled |
| Success | Green "Settings saved" text appears, fades after 3s |
| Error | Toast: "Failed to save cache settings. Try again." |
| No Redis | If Redis is not configured, show info banner: "Response caching requires Redis. Set the REDIS_URL environment variable to enable caching." Banner style: `bg-blue-900/20 border border-blue-500/30 rounded-lg p-4 text-sm text-zinc-400` |

#### SmartRoutingToggle Component

Props:
- `enabled: boolean` — bindable, default `false`
- `orgHasRouting: boolean` — whether org has smart routing models configured

Layout:
- Container: `flex items-start gap-3 rounded-lg border border-zinc-800 bg-zinc-900/50 p-4 mt-4`
- Toggle switch: 44px wide, 24px tall, rounded-full
  - Off: `bg-zinc-700` track, white knob left-aligned
  - On: `bg-blue-600` track, white knob right-aligned
  - Transition: `transition-colors duration-200`
  - Knob: `w-5 h-5 bg-white rounded-full shadow transition-transform duration-200`
- Label: `text-sm font-bold text-zinc-200` — "Enable smart routing"
- Description: `text-xs text-zinc-500 mt-1` — "Automatically route simple queries to cheaper models."
- Hidden input: `name="smartRouting"` with value "true" or "false"

States:

| State | Behavior |
|-------|----------|
| Off (default) | Toggle off, description shown |
| On | Toggle on (blue), description shown |
| Org not configured | Toggle disabled (`opacity-50 cursor-not-allowed`), description changes to "Configure smart routing models in organization settings first." |

Accessibility:
- Toggle: `role="switch"` `aria-checked={enabled}` `aria-label="Enable smart routing"`
- Keyboard: `Space` or `Enter` toggles the switch
- Focus ring: `focus-visible:ring-2 focus-visible:ring-blue-500 focus-visible:ring-offset-2 focus-visible:ring-offset-zinc-900`

## Interaction Patterns

### Landing Page Navigation (authenticated vs unauthenticated)

- **Unauthenticated:** "Log In" and "Get Started" buttons visible. Hero CTA says "Get Started Free".
- **Authenticated:** Both buttons replaced with "Go to Dashboard" linking to `/org/{slug}/dashboard`. Hero CTA also says "Go to Dashboard".
- Auth check happens in `+page.server.ts` via `locals.session`. If session exists and user has an org, redirect to `/org/{slug}/dashboard` (server-side redirect, user never sees landing page).

### Smart Routing Settings Save Flow

1. Admin changes cheap or expensive model text input
2. Clicks "Save Routing Settings"
3. Form submits via SvelteKit form action with `use:enhance`
4. On success: "Settings saved" green text appears, fades after 3s
5. On failure: toast notification
6. Both fields can be emptied to disable smart routing at the org level

### Cache TTL Save Flow

1. Admin changes the TTL number input
2. Clicks "Save Cache Settings"
3. Same success/error pattern as smart routing
4. Input validates: minimum 60, maximum 86400 (24 hours)
5. Invalid values show inline error: "TTL must be between 60 and 86,400 seconds."

### Smart Routing Toggle on API Key Creation

1. User opens Create API Key modal
2. Toggle defaults to off
3. If org has smart routing configured, toggle is interactive
4. If org does NOT have smart routing configured, toggle is disabled with explanation text
5. On key creation, `smartRouting` boolean is submitted with the form
6. Toggle state is visual-only — no save action until the key is created

### Integration Docs Tab Switching

1. User clicks a tool tab (Cursor, Continue.dev, Claude Code)
2. Content below updates to show that tool's guide
3. URL hash updates (no page reload): `#cursor`, `#continue`, `#claude-code`
4. On page load, if URL has a hash, that tab is selected
5. Default: Cursor tab

### Code Block Copy

1. User clicks "Copy" button on a code block
2. Text copied to clipboard via `navigator.clipboard.writeText()`
3. Button text changes to "Copied!" in green for 2 seconds
4. Reverts to "Copy" after timeout

### Loading States

- Landing page: static content, no loading states needed (server-rendered)
- Settings forms: save button shows "Saving..." with spinner (`animate-spin`), inputs disabled
- Integration docs: static content, no loading states needed

### Error States

- Settings save failed: toast notification `bg-red-900/20 border border-red-500/30 text-red-400 text-sm rounded-lg px-4 py-3`, auto-dismiss after 5s
- No Redis configured (cache settings): info banner within the section (not blocking — admin can still set the TTL for when Redis is eventually configured)

## Accessibility

All interactive elements must include appropriate aria attributes:

| Element | Attribute |
|---------|-----------|
| Landing nav | `<nav aria-label="Main navigation">` |
| Landing hero CTA | Standard `<a>` element with visible text — no extra aria needed |
| Feature cards | Each card is a `<div>` (not interactive) — no aria needed. Static component — no interactive states. |
| Cost comparison | Structured as a semantic `<section>` with heading. Static component — no interactive states. |
| Smart routing toggle | `role="switch"` `aria-checked={enabled}` `aria-label="Enable smart routing"` |
| Smart routing toggle (disabled) | `aria-disabled="true"` added |
| Cache TTL input | `aria-label="Cache time to live in seconds"` |
| Cheap model input | `aria-label="Cheap model for simple queries"` |
| Expensive model input | `aria-label="Expensive model for complex queries"` |
| Tool tabs | `role="tablist"` with `role="tab"` children, `aria-selected` on active tab |
| Tab panel | `role="tabpanel"` `aria-labelledby="{tab-id}"` |
| Code block copy button | `aria-label="Copy code to clipboard"` |
| Code block copy (after) | `aria-live="polite"` on the "Copied!" text |
| Landing nav links | Standard `<a>` elements — no extra aria needed |
| Save buttons | Standard `<button type="submit">` — button text communicates action |
| Disabled toggle | `tabindex="-1"` to skip in tab order when disabled |
