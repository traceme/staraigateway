## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding |
| sm | 8px | Compact element spacing, table cell padding vertical |
| md | 16px | Default element spacing, card padding, table cell padding horizontal |
| lg | 24px | Section padding, card internal sections |
| xl | 32px | Layout gaps, page section breaks |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level spacing |

Exceptions: Chart containers use 24px internal padding. Tab bar uses 8px gap between tab items.

Source: Derived from existing component patterns (`px-4` = 16px, `py-3` = 12px in tables, `space-y-6` = 24px, `mb-8` = 32px).

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 14px | 400 (regular) | 1.5 |
| Label | 12px | 400 (regular) | 1.4 |
| Heading | 24px | 700 (bold) | 1.2 |
| Stat display | 32px | 700 (bold) | 1.1 |

Notes:
- Body (`text-sm`) is the dominant size across tables, descriptions, and chart labels.
- Label (`text-xs`) is used for table headers, badges, axis labels, and secondary metadata. Uses weight 400 (regular); table headers achieve visual distinction via `uppercase tracking-wider` rather than a heavier weight.
- Heading (`text-2xl font-bold`) matches the existing page title pattern from dashboard and api-keys pages.
- Stat display (`text-3xl font-bold`) is new for this phase — used in KPI summary cards (total spend, request count).

Source: Existing patterns use `text-2xl font-bold` for page titles, `text-sm` for body, `text-xs uppercase tracking-wider` for table headers.

### Chart Library Decision

Use **Chart.js** (`chart.js` + `svelte-chartjs` or direct Canvas API) for charting:
- Lightweight (~60KB gzipped for line + bar)
- Canvas-based (performant for daily data points across 30-90 day ranges)
- No React dependency (works natively with Svelte via canvas element)
- Sufficient for line charts + bar charts without D3 complexity

Alternative if `svelte-chartjs` has compatibility issues with Svelte 5: use Chart.js directly via `<canvas>` element with `onMount` lifecycle.

## Interaction Patterns

### Tab Navigation (Usage page)
- Tabs: Overview, By Member, By Model
- Active tab: `text-zinc-50` with 2px `border-b-2 border-blue-500` underline
- Inactive tab: `text-zinc-400 hover:text-zinc-300`
- Tab switch re-renders chart and table content; no full page reload (client-side state)
- URL updates with `?tab=overview|member|model` for shareability

### Time Range Selection
- Quick toggles: pill-shaped buttons `[7d]` `[30d]`
- Active toggle: `bg-zinc-700 text-zinc-100`
- Inactive toggle: `bg-transparent text-zinc-400 hover:text-zinc-300`
- Custom range: opens a minimal date picker (native `<input type="date">` pair) — no third-party date picker library
- Selected range persists across tab switches within the same session

### Sortable Tables
- Click column header to sort ascending; click again for descending
- Sort indicator: small arrow icon (▲/▼) next to active sort column header
- Default sort: cost descending (highest spender/model first)
- Table headers use existing pattern: `text-xs uppercase tracking-wider text-zinc-500`

### Budget Panel (slide-out)
- Trigger: "Set Budget" button on member row in the members list (or usage by-member tab)
- Animation: slide from right, 200ms ease-out
- Close: click X (`aria-label="Close"`), click backdrop, or press Escape
- Save: SvelteKit form action with `use:enhance` (matches existing pattern)
- Validation: hard limit must be >= soft limit (if both set), minimum $1

### Chart Interactions
- Hover on data point: tooltip showing date + exact cost value
- Tooltip style: `bg-zinc-800 border border-zinc-700 rounded-md px-3 py-2 text-sm`
- No click-to-drill-down (keep it simple for Phase 3)
- Charts resize responsively with container

### Loading States
- Charts: skeleton placeholder — pulsing `bg-zinc-800/50` rectangle at chart dimensions (280px height)
- KPI cards: skeleton with pulsing value placeholder
- Tables: 3 skeleton rows with pulsing cell placeholders
- All skeletons use `animate-pulse` (existing Tailwind utility)

### Error States
- Chart load failure: replace chart area with centered error message + retry button
- Error card: `bg-red-900/20 border border-red-800/30 rounded-lg p-6`
- Copy: "Failed to load usage data" + "Try again" button

## Email Templates

### Budget Warning Email (to member)

Subject: "You're approaching your monthly AI budget"

Body structure:
- Greeting with member name
- Current spend vs limit (e.g., "$90.00 of $100.00")
- Visual progress indicator (HTML table-based bar, not CSS — for email compatibility)
- "Contact your admin if you need a higher limit"
- Organization name in footer

### Admin Daily Digest Email

Subject: "LLMTokenHub: Daily budget digest for {org name}"

Body structure:
- Date header
- Table of members approaching/exceeding limits: Name, Spend, Limit, Percentage
- Only includes members at 90%+ threshold
- "Manage budgets" link to budget settings page
