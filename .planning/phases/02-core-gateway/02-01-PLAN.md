1. **Add two new tables to schema.ts** (keep all existing tables unchanged):

   `appProviderKeys` table (app_provider_keys):
   - id: text PK (use crypto.randomUUID())
   - orgId: text FK -> appOrganizations.id, NOT NULL
   - provider: text NOT NULL (e.g., 'openai', 'anthropic', 'google', 'deepseek', etc.)
   - label: text NOT NULL (e.g., 'Production', 'Dev team')
   - encryptedKey: text NOT NULL (stores IV:ciphertext:authTag as hex)
   - baseUrl: text (nullable, for custom OpenAI-compatible endpoints)
   - models: text (nullable, JSON string of discovered model names from validation)
   - isActive: boolean NOT NULL default true
   - createdAt, updatedAt: timestamps with timezone

   Index on (orgId, provider). Unique constraint on (orgId, provider, label).

   `appApiKeys` table (app_api_keys):
   - id: text PK
   - orgId: text FK -> appOrganizations.id, NOT NULL
   - userId: text FK -> appUsers.id, NOT NULL
   - name: text NOT NULL (e.g., 'Cursor', 'Claude Code CLI')
   - keyPrefix: text NOT NULL (e.g., 'sk-th-abc123') -- first 12 chars for display
   - keyHash: text NOT NULL (SHA-256 hash of full key)
   - isActive: boolean NOT NULL default true
   - lastUsedAt: timestamp nullable
   - createdAt: timestamp with timezone

   Index on keyHash (for lookup). Index on (orgId, userId).

2. **Update src/lib/types/index.ts**: Add inferred types for both new tables (ProviderKey, ApiKey, NewProviderKey, NewApiKey).

3. **Create src/lib/server/crypto.ts**: AES-256-GCM encryption module.
   - Read ENCRYPTION_KEY from `$env/dynamic/private` (32-byte hex string = 64 hex chars)
   - `encrypt(plaintext: string): string` -- generates random 12-byte IV, encrypts, returns `${iv_hex}:${ciphertext_hex}:${authTag_hex}`
   - `decrypt(encrypted: string): string` -- splits on ':', decrypts with IV and auth tag
   - Use Node.js built-in `crypto` module (createCipheriv/createDecipheriv with 'aes-256-gcm')
   - Throw descriptive error if ENCRYPTION_KEY is not set or wrong length

4. **Create src/lib/server/providers.ts**: Provider catalog definition.
   - Export `PROVIDERS` array with two groups: 'global' and 'china'
   - Each provider: { id: string, name: string, group: 'global'|'china', modelsEndpoint: string, authHeader: string, docsUrl: string, baseUrl: string, icon?: string }
   - Global providers: openai, anthropic, google, azure, mistral, cohere
   - China providers: deepseek, qwen, glm, doubao
   - Include 'custom' provider with group 'custom' for OpenAI-compatible endpoints
   - modelsEndpoint is the URL path to list models (e.g., '/v1/models' for OpenAI-compatible, specific URLs for others)
   - authHeader format: 'Bearer' for most, 'x-api-key' for Anthropic
   - docsUrl links to each provider's API key page

5. **Update .env.example**: Add ENCRYPTION_KEY with a comment explaining it must be 64 hex chars (32 bytes).

  <name>Task 2: Provider key CRUD + validation server logic</name>
  <files>src/lib/server/provider-keys.ts, src/routes/org/[slug]/provider-keys/+page.server.ts, src/routes/org/[slug]/provider-keys/validate/+server.ts</files>
  <action>
1. **Create src/lib/server/provider-keys.ts**: CRUD operations for provider keys.
   - `getProviderKeys(orgId: string)`: Select all keys for org, return WITHOUT decrypting (just metadata: id, provider, label, baseUrl, models, isActive, createdAt). Never expose encrypted key value to client.
   - `createProviderKey(orgId: string, data: { provider, label, apiKey, baseUrl? })`: Encrypt apiKey with crypto.encrypt(), insert row, return the created key metadata.
   - `updateProviderKey(id: string, orgId: string, data: { label?, apiKey?, baseUrl?, isActive? })`: If apiKey provided, encrypt it. Update only provided fields. Verify key belongs to orgId before updating.
   - `deleteProviderKey(id: string, orgId: string)`: Delete key, verify belongs to orgId.
   - `validateProviderKey(provider: string, apiKey: string, baseUrl?: string)`: Call the provider's /models endpoint with the given key. Return { valid: boolean, models: string[], error?: string }. Use the PROVIDERS catalog to determine the correct endpoint and auth header format. For custom providers, use the provided baseUrl.
   - `decryptProviderKeyById(id: string, orgId: string)`: Look up key by id+orgId, decrypt and return the plaintext API key. Used by the gateway (Plan 03).

2. **Create src/routes/org/[slug]/provider-keys/+page.server.ts**: Page load + form actions.
   - `load`: Check user role is 'owner' or 'admin' (not 'member'). Load all provider keys for org via getProviderKeys(). Return { providerKeys, providers: PROVIDERS }.
   - Form actions:
     - `create`: Validate with Zod (provider required, label required 1-50 chars, apiKey required, baseUrl optional valid URL). Call createProviderKey().
     - `update`: Validate id + fields. Call updateProviderKey().
     - `delete`: Validate id. Call deleteProviderKey().
   - All actions check role is owner/admin, return fail(403) for members.

3. **Create src/routes/org/[slug]/provider-keys/validate/+server.ts**: POST endpoint for key validation.
   - Accepts JSON { provider, apiKey, baseUrl? }
   - Calls validateProviderKey() and returns JSON { valid, models, error? }
   - Checks session + org membership + role (owner/admin only)
   - This is a +server.ts (API endpoint), not a form action, because validation is async and should not reload the page.
  </action>
