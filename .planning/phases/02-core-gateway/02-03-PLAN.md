  <action>
1. **Create src/lib/server/gateway/auth.ts**: API key authentication for gateway requests.
   - `authenticateApiKey(request: Request): Promise<{ userId: string, orgId: string, org: OrgRow } | null>`
   - Extract Bearer token from Authorization header (strip 'Bearer ' prefix)
   - If no token or doesn't start with 'sk-th-', return null
   - Hash the token with SHA-256 (same algorithm as api-keys.ts uses)
   - Look up in appApiKeys by keyHash where isActive = true
   - Join with appUsers and appOrganizations to get user and org data
   - Update lastUsedAt timestamp on the key
   - Return { userId, orgId, org } or null if not found/inactive
   - NOTE: This duplicates the hash+lookup logic from api-keys.ts validateApiKeyFromHash intentionally -- Plan 02 and Plan 03 run in parallel, so this plan cannot import from Plan 02. Once both plans are complete, a future cleanup can consolidate. Import directly from schema and db.

2. **Create src/lib/server/gateway/proxy.ts**: Stream-proxy to LiteLLM.
   - `proxyToLiteLLM(request: Request, orgId: string, path: string): Promise<Response>`
   - Determine the requested model from the request body (parse JSON for model field)
   - Find a matching active provider key for this org: query appProviderKeys where orgId matches, isActive, and the models JSON array contains the requested model name. Use the first matching key (simple selection per CONTEXT.md -- "first active key that supports the requested model wins").
   - If no provider key found for the model, return 404 JSON: { error: { message: "No provider configured for model: {model}", type: "invalid_request_error" } }
   - Decrypt the provider key using crypto.decrypt()
   - Forward the request to LiteLLM at `${LITELLM_API_URL}${path}` with:
     - The decrypted provider key in the Authorization header (using the correct format from PROVIDERS catalog -- Bearer for most, x-api-key for Anthropic)
     - Also pass the provider's base URL if it's a custom provider
     - Pass through the original request body unchanged (preserves tool_choice, tools, functions, stream, etc.)
     - Pass through Content-Type header
   - For streaming responses (when body contains `"stream": true`):
     - Return the LiteLLM response directly as a streaming Response with the same headers (Content-Type: text/event-stream)
     - Do NOT buffer or transform the SSE stream -- pure pass-through
   - For non-streaming responses:
     - Return the JSON response from LiteLLM directly
   - On LiteLLM error, return the error with appropriate status code

3. **Create src/lib/server/gateway/models.ts**: Model aggregation for /v1/models.
   - `getAvailableModels(orgId: string): Promise<{ id: string, object: string, owned_by: string }[]>`
   - Query all active provider keys for the org
   - Parse the models JSON from each key
   - Aggregate into a deduplicated list in OpenAI /v1/models response format
   - Each model: { id: "gpt-4o", object: "model", created: timestamp, owned_by: "openai" }
   - owned_by = the provider name from the provider key

4. **Update src/hooks.server.ts**: Skip session cookie validation for /v1/* routes.
   - Add a check at the top: if `event.url.pathname.startsWith('/v1/')`, skip the session cookie logic entirely (return resolve(event) immediately with locals.user = null, locals.session = null)
   - /v1/* routes use API key auth (Bearer token), not session cookies
   - Keep all existing session logic unchanged for non-/v1/ routes
