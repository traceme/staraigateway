Create `src/lib/server/auth/password.ts`:
- `hashPassword(password: string): Promise<string>` using `@node-rs/argon2` with recommended params (memoryCost: 19456, timeCost: 2, outputLen: 32, parallelism: 1)
- `verifyPassword(hash: string, password: string): Promise<boolean>`

Create `src/lib/server/auth/session.ts` (~200 lines, Lucia pattern per user decision):
- `generateSessionToken(): string` -- use `crypto.getRandomValues()` + base32 encoding (32 bytes)
- `createSession(userId: string): Promise<{ token: string; session: Session }>` -- insert into app_sessions with 30-day expiry, return token (unhashed) and session
- `validateSession(token: string): Promise<{ user: User; session: Session } | null>` -- hash token with SHA-256, look up in app_sessions joined with app_users, check expiry. If session expires within 15 days, extend it (sliding window). Return null if expired/not found.
- `invalidateSession(sessionId: string): Promise<void>` -- delete from app_sessions
- `invalidateAllUserSessions(userId: string): Promise<void>` -- delete all sessions for user
- Session token stored in DB as SHA-256 hash (the cookie holds the unhashed token, DB stores hashed version for security)

Create `src/lib/server/auth/email.ts`:
- Configure Nodemailer transport using SMTP_HOST, SMTP_PORT, SMTP_USER, SMTP_PASS from env
- `sendVerificationEmail(email: string, name: string, token: string): Promise<void>` -- sends email with link `${APP_URL}/auth/verify-email?token=${token}`
- `sendPasswordResetEmail(email: string, name: string, token: string): Promise<void>` -- sends email with link `${APP_URL}/auth/reset-password?token=${token}`
- Use the HTML templates from the emails/ subfolder

Create `src/lib/server/auth/emails/verification.ts`:
- Export function that returns clean HTML email: subject "Verify your email", body with verification link button, plain text fallback
- Minimal styling: white background, centered card, blue button, clean typography

Create `src/lib/server/auth/emails/password-reset.ts`:
- Export function that returns clean HTML email: subject "Reset your password", body with reset link button, expiry warning (1 hour)
- Same minimal styling as verification email

Create `src/lib/server/auth/validation.ts`:
- Zod schemas: `signupSchema` (email, password min 8 chars, name min 1), `loginSchema` (email, password), `forgotPasswordSchema` (email), `resetPasswordSchema` (token, password min 8)
- Export all schemas

Create `src/hooks.server.ts`:
- On every request, read session token from `auth_session` cookie
- Call `validateSession(token)` -- if valid, set `event.locals.user` and `event.locals.session`
- If session was extended (sliding window), update the cookie
- If no valid session, set locals to null

Create signup flow (`src/routes/auth/signup/`):
- `+page.server.ts`: SvelteKit form action that:
  1. Validates input with signupSchema (zod)
  2. Checks email not already taken (query app_users)
  3. Hashes password with Argon2id
  4. Inserts user into app_users (email_verified: false)
  5. Generates verification token (crypto.randomUUID()), inserts into app_email_verifications with 24h expiry
  6. Sends verification email
  7. Returns success message: "Check your email to verify your account"
  8. On error: return form errors via SvelteKit `fail()`
- `+page.svelte`: Form with name, email, password fields. Show form errors inline. Link to login page. Show success message after submit.

Create login flow (`src/routes/auth/login/`):
- `+page.server.ts`: Form action that:
  1. Validates with loginSchema
  2. Looks up user by email
  3. Verifies password with Argon2id
  4. Checks email_verified is true -- if not, return error "Please verify your email first"
  5. Creates session, sets `auth_session` cookie (httpOnly, secure in prod, sameSite: lax, path: /, maxAge: 30 days)
  6. Redirects to `/` (will redirect to org creation or dashboard in Plan 03)
- `+page.svelte`: Form with email, password fields. Error messages. Links to signup and forgot-password.

  2. Looks up in app_email_verifications, checks not expired
  3. Updates app_users set email_verified = true
  4. Deletes the verification token
  5. Returns success or error
- `+page.svelte`: Shows "Email verified! You can now log in." with link to login, or error message if token invalid/expired.
