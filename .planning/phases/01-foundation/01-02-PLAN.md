    - verifyPassword(hash, "test123") returns true for matching password
    - verifyPassword(hash, "wrong") returns false
    - createSession(userId) returns a session object with id, userId, expiresAt (30 days from now)
    - validateSession(validToken) returns { session, user } with user data
    - validateSession(expiredToken) returns { session: null, user: null }
    - invalidateSession(sessionId) removes it from DB
    - generateVerificationToken(userId) creates token expiring in 24h
    - verifyEmailToken(validToken) marks user email_verified=true and deletes token
    - verifyEmailToken(expiredToken) returns error
    - generatePasswordResetToken(email) creates token expiring in 1h
    - resetPassword(validToken, newPassword) updates password hash, invalidates token
  </behavior>
  <action>
    1. Create `src/lib/server/auth/password.ts`:
       - `hashPassword(password: string): Promise<string>` -- Argon2id with recommended params (per user decision)
       - `verifyPassword(hash: string, password: string): Promise<boolean>`

    2. Create `src/lib/server/auth/session.ts` following Lucia patterns (~200 lines per user decision):
       - `createSession(userId: string): Promise<Session>` -- insert into app_sessions with nanoid token, 30-day expiry, return session object
       - `validateSession(sessionId: string): Promise<SessionValidationResult>` -- look up session, check expiry, extend if within 15 days of expiry (sliding window), return { session, user } or { session: null, user: null }
       - `invalidateSession(sessionId: string): Promise<void>` -- delete from app_sessions
       - `setSessionCookie(cookies: Cookies, sessionId: string): void` -- set HTTP-only, secure, SameSite=lax cookie named "session"
       - `deleteSessionCookie(cookies: Cookies): void`

    3. Create `src/lib/server/auth/email.ts`:
       - Initialize Nodemailer transport from env vars (SMTP_HOST, SMTP_PORT, SMTP_USER, SMTP_PASS)
       - `sendVerificationEmail(email: string, token: string): Promise<void>` -- sends link to `${APP_URL}/auth/verify-email?token={token}`
       - `sendPasswordResetEmail(email: string, token: string): Promise<void>` -- sends link to `${APP_URL}/auth/reset-password?token={token}`
       - Clean, minimal HTML templates (per user decision: functional, not fancy)

    4. Create `src/lib/server/auth/verification.ts`:
       - `generateVerificationToken(userId: string): Promise<string>` -- insert into app_email_verifications, token via nanoid, expires in 24h
       - `verifyEmailToken(token: string): Promise<{ success: boolean; error?: string }>` -- validate token, mark user verified, delete token
       - `generatePasswordResetToken(email: string): Promise<string | null>` -- find user by email, insert into app_password_resets, token via nanoid, expires in 1h. Return null if user not found (don't leak user existence).
       - `resetPassword(token: string, newPassword: string): Promise<{ success: boolean; error?: string }>` -- validate token, hash new password, update user, mark token used, invalidate all user sessions

    5. Create `src/lib/server/auth/index.ts` barrel file exporting all public functions.

    1. Create `src/hooks.server.ts`:
       - On every request, read "session" cookie
       - If present, call `validateSession()` and set `event.locals.user` and `event.locals.session`
       - If expired or invalid, delete cookie

    2. Create route handlers:

       **POST /auth/signup** (`+page.server.ts` actions):
       - Validate email format and password length (min 8 chars)
       - Check email not already registered
       - Hash password, insert user into app_users
       - Generate verification token, send verification email
       - Return success message: "Check your email to verify your account"
       - Do NOT create session yet (per user decision: email verification required first)

       **GET /auth/verify-email** (`+page.server.ts` load):
       - Read `token` from URL params
       - Call verifyEmailToken -- on success show "Email verified, you can now log in"
       - On failure show "Invalid or expired link"

       **POST /auth/login** (`+page.server.ts` actions):
       - Find user by email, verify password
       - If user not email_verified, return error "Please verify your email first"
       - Create session, set session cookie, redirect to `/` (root will redirect to org page)

       **POST /auth/forgot-password** (`+page.server.ts` actions):
       - Accept email, call generatePasswordResetToken
       - Always return "If an account exists, we sent a reset link" (prevent enumeration)
       - Send email if user found

       **POST /auth/reset-password** (`+page.server.ts` actions):
       - Read token from URL params, new password from form
       - Call resetPassword -- on success redirect to /auth/login with flash message
       - On failure show "Invalid or expired reset link"

       **POST /auth/logout** (`+page.server.ts` actions):
       - Call invalidateSession and deleteSessionCookie, redirect to /auth/login

    3. Create Svelte pages for each auth route:
       - Clean, centered card layout (think Linear/Vercel login page per user specifics)
       - Minimal: logo area, form fields, submit button, link to alternate action
       - Use Tailwind classes directly -- no component library yet
       - Show loading state on submit, display server errors inline
       - Signup page: email, password, confirm password fields + "Already have an account? Log in" link
       - Login page: email, password fields + "Forgot password?" link + "Create account" link
       - Forgot password page: email field + "Back to login" link
       - Reset password page: new password, confirm password fields
