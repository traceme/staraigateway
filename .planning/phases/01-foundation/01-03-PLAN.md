Create `src/lib/server/litellm.ts` -- LiteLLM management API client:
- Read `LITELLM_API_URL` and `LITELLM_MASTER_KEY` from env
- `createLiteLLMOrganization(name: string, alias: string): Promise<{ organization_id: string } | null>`:
  - POST to `${LITELLM_API_URL}/organization/new` with body `{ organization_alias: alias }` and header `Authorization: Bearer ${LITELLM_MASTER_KEY}`
  - Return the organization_id from response
  - On failure (connection error, 4xx, 5xx): log warning and return null (don't block org creation if LiteLLM is unavailable -- we'll store null as litellm_org_id and retry later)
- `checkLiteLLMHealth(): Promise<boolean>`:
  - GET `${LITELLM_API_URL}/health`
  - Return true if 200, false otherwise

Create `src/routes/+page.server.ts` -- Root page redirect logic:
- Load function checks `event.locals.user`:
  - Not logged in -> redirect to `/auth/login`
  - Logged in -> query app_org_members for user's orgs
  - Has orgs -> redirect to `/org/${firstOrg.slug}/dashboard`
  - No orgs -> redirect to `/org/create`

Create `src/routes/org/create/+page.server.ts`:
- Load: Require auth (redirect to login if not logged in)
- Action:
  1. Validate: name (required, 2-50 chars), description (optional, max 200 chars)
  2. Generate slug from name: lowercase, replace spaces with hyphens, remove non-alphanumeric except hyphens, trim hyphens. If slug already taken, append random 4-char suffix.
  3. Generate org ID with crypto.randomUUID()
  4. Call `createLiteLLMOrganization(name, slug)` -- store returned org_id or null
  5. Insert into app_organizations
  6. Insert into app_org_members (user as 'owner')
  7. Redirect to `/org/${slug}/dashboard`

- Fields: Organization name (required), Description (optional textarea)
- "Create Organization" primary button

- Show validation errors inline
- Page title: "Create Organization"
