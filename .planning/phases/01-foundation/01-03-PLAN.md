  <action>
    1. Create `src/lib/server/org/index.ts` with organization operations:

       - `createOrganization(params: { name: string; description?: string; userId: string }): Promise<Organization>`
         - Generate slug from name: lowercase, replace spaces/special chars with hyphens, trim to 48 chars, append 4-char nanoid suffix to ensure uniqueness (e.g., "Acme Corp" -> "acme-corp-x7k2")
         - Insert into app_organizations
         - Insert into app_org_members with role='owner' and the creating user's ID
         - Return the created org object

       - `getUserOrganizations(userId: string): Promise<OrgWithRole[]>`
         - Join app_org_members with app_organizations where user_id matches
         - Return array of { org, role } objects
         - Used by the layout to populate org switcher

       - `getOrgBySlug(slug: string): Promise<Organization | null>`
         - Simple lookup by slug

       - `isOrgMember(orgId: string, userId: string): Promise<boolean>`
         - Check if user has any role in the org

    2. Create `src/lib/server/litellm.ts` -- LiteLLM management API client:

       - Read `LITELLM_API_URL` (default: `http://localhost:4000`) and `LITELLM_MASTER_KEY` from env
       - `createLiteLLMOrganization(params: { name: string; orgId: string }): Promise<string | null>`
         - POST to `${LITELLM_API_URL}/organization/new` with body `{ organization_alias: name, metadata: { app_org_id: orgId } }`
         - Authorization header: `Bearer ${LITELLM_MASTER_KEY}`
         - Return the LiteLLM org ID on success, null on failure
         - Log warning but do NOT fail org creation if LiteLLM is unavailable (graceful degradation -- LiteLLM might not be running during dev)
         - Update app_organizations.litellm_org_id with the returned ID

    3. Create org creation route `src/routes/(app)/org/create/+page.server.ts`:
       - `load`: If user already has orgs, still allow creating more (multi-org per user decision)
       - `actions.default`: Validate name (3-50 chars), call createOrganization, attempt createLiteLLMOrganization, redirect to `/org/${slug}/dashboard`

    4. Create `src/routes/(app)/org/create/+page.svelte`:
       - Clean centered card (consistent with auth pages)
       - Fields: Organization name (required), Description (optional textarea)
       - Submit button: "Create Organization"
       - Show validation errors inline

    5. Create `src/routes/+page.server.ts` (root route):
       - If not authenticated: redirect to /auth/login
       - If authenticated but no orgs: redirect to /org/create
       - If authenticated with orgs: redirect to /org/{first-org-slug}/dashboard

    6. Add `LITELLM_API_URL` and `LITELLM_MASTER_KEY` to `.env.example`.
