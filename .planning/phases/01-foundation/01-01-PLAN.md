Create `src/lib/server/db/schema.ts` with all tables using `app_` prefix per user decision (to coexist with LiteLLM's Prisma tables):

1. `app_users` table:
   - `id` text primary key (use `crypto.randomUUID()` default or generate in app code)
   - `email` text not null unique
   - `password_hash` text not null
   - `name` text not null
   - `email_verified` boolean not null default false
   - `created_at` timestamp not null default now()
   - `updated_at` timestamp not null default now()

2. `app_sessions` table:
   - `id` text primary key (session token)
   - `user_id` text not null references app_users
   - `expires_at` timestamp with timezone not null
   - `created_at` timestamp not null default now()

3. `app_organizations` table:
   - `id` text primary key
   - `name` text not null
   - `slug` text not null unique (URL-friendly)
   - `description` text (nullable)
   - `litellm_org_id` text (nullable -- populated after LiteLLM API call)
   - `created_at` timestamp not null default now()
   - `updated_at` timestamp not null default now()

4. `app_org_members` table:
   - `id` text primary key
   - `org_id` text not null references app_organizations
   - `user_id` text not null references app_users
   - `role` text not null (enum: 'owner', 'admin', 'member') -- use a check constraint or just store text
   - `created_at` timestamp not null default now()
   - Unique constraint on (org_id, user_id)

5. `app_email_verifications` table:
   - `id` text primary key (the token)
   - `user_id` text not null references app_users
   - `expires_at` timestamp with timezone not null
   - `created_at` timestamp not null default now()

6. `app_password_resets` table:
   - `id` text primary key (the token)
   - `user_id` text not null references app_users
   - `expires_at` timestamp with timezone not null
   - `created_at` timestamp not null default now()

Add proper indexes: email on users, slug on orgs, user_id on sessions, (org_id, user_id) on members.
