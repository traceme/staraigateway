Create `src/lib/server/gateway/rate-limit.test.ts` co-located with the source module. Use `import { describe, it, expect, vi, beforeEach } from 'vitest'` and import all 5 exports from `./rate-limit`.

No external mocks needed — rate-limit is pure in-memory logic. However, the module has a `setInterval(cleanup, 60_000)` at module level that will run in tests. Use `vi.useFakeTimers()` in a `beforeEach` and `vi.useRealTimers()` in an `afterEach` to control time.

Between each test that mutates the sliding window, call `cleanup()` or use fresh keyId strings to avoid cross-test state leakage.

Test cases for `checkRateLimit`:
1. **No limits (null, null)** — returns `{ allowed: true, rpmLimit: null, tpmLimit: null }`
2. **Under RPM limit** — 5 requests with rpmLimit=10 returns `{ allowed: true, rpmRemaining: 5 }`
3. **RPM limit exactly hit** — 10 requests with rpmLimit=10 returns `{ allowed: false, rpmRemaining: 0 }`
4. **TPM limit exceeded** — record 1000 tokens then check with tpmLimit=500, returns `{ allowed: false, tpmRemaining: 0 }`
5. **Both limits set, RPM triggers first** — rpmLimit=2, tpmLimit=10000, make 2 requests → allowed: false
6. **Stale entries evicted** — advance time by 61 seconds via `vi.advanceTimersByTime(61000)`, entries should be gone, request allowed again

Test cases for `recordRequest`:
7. **Records entry** — after `recordRequest('key1', 100)`, `checkRateLimit('key1', 10, null)` shows 1 request counted

Test cases for `cleanup`:
8. **Removes stale entries** — add entries, advance time 61s, call `cleanup()`, then `checkRateLimit` shows 0 current usage

Test cases for `rateLimitResponse`:
9. **Returns 429 with correct body** — pass a RateLimitResult with rpmLimit=10, rpmRemaining=0, verify response.status === 429, body contains `"rate_limit_exceeded"`, headers include `Retry-After` and `x-ratelimit-limit-requests`
10. **Only includes TPM headers when tpmLimit set** — rpmLimit=null, tpmLimit=1000, verify `x-ratelimit-limit-tokens` present but `x-ratelimit-limit-requests` absent

Test cases for `addRateLimitHeaders`:
11. **Returns original response when no limits** — rpmLimit=null, tpmLimit=null, returned response is the same object
12. **Adds RPM headers to response** — rpmLimit=100, verify `x-ratelimit-limit-requests` = "100" on returned response
13. **Preserves original response body and status** — original response has status 200 and body "ok", returned response has same status and body
