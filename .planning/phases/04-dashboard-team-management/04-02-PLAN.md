1. Create `src/lib/server/members.ts` with these exported functions:
   - `inviteMember(orgId: string, email: string, role: 'admin' | 'member', invitedByUserId: string)`: Generate random token (crypto.randomUUID()), insert into appOrgInvitations with 7-day expiresAt, call sendInvitationEmail. Check for existing member (error: "already a member") and existing pending invite (error: "already invited").
   - `acceptInvitation(token: string, userId: string)`: Look up invitation by token, check not expired and not already accepted. Insert into appOrgMembers with the invitation's role. Update invitation acceptedAt. Also create a LiteLLM org membership if applicable (follow pattern from 01-03).
   - `removeMember(orgId: string, targetUserId: string, actorRole: string)`: Check actor is admin/owner. Cannot remove the owner. Delete from appOrgMembers. Deactivate all of the removed user's API keys for this org (set isActive=false on appApiKeys where userId=targetUserId and orgId matches).
   - `changeRole(orgId: string, targetUserId: string, newRole: 'admin' | 'member', actorRole: string)`: Check actor is owner (only owner can change roles). Cannot change owner's own role. Update appOrgMembers.role.
   - `revokeInvitation(orgId: string, invitationId: string)`: Delete the invitation row from appOrgInvitations.

2. Create `src/lib/server/auth/emails/invitation.ts`:
   - Follow existing verification.ts email template pattern (table-based HTML, blue CTA button #2563eb).
   - Subject: "You've been invited to join {orgName} on LLMTokenHub"
   - Body: greeting, "{inviterName} has invited you to join {orgName} as a {role}.", CTA button "Accept Invitation" linking to `{appUrl}/auth/invite/{token}`, expiry note "This invitation expires in 7 days."

3. Add `sendInvitationEmail(email, token, orgName, inviterName, role)` to `src/lib/server/auth/email.ts` following the existing sendVerificationEmail pattern with graceful SMTP failure handling.

4. Create `src/routes/auth/invite/[token]/+page.server.ts`:
   - Load function: look up invitation by token. If not found or expired, return `{ valid: false }`. If valid, return `{ valid: true, orgName, role, inviterName }`.
   - If user is logged in (locals.user exists): show accept/decline buttons.
   - Form action `accept`: call `acceptInvitation(token, locals.user.id)`, redirect to org dashboard.
   - Form action `decline`: delete the invitation, redirect to '/'.
   - If user is NOT logged in: redirect to `/auth/login?redirect=/auth/invite/${token}` so they can log in first, then come back.

5. Create `src/routes/auth/invite/[token]/+page.svelte`:
   - Show invitation details (org name, role, inviter) with accept/decline buttons.
   - If invalid/expired: show "This invitation is no longer valid" message.
   - Use `bg-zinc-950` page background, centered card layout like login page.
  </action>
