   - Change `appUsers.passwordHash` from `.notNull()` to nullable (remove `.notNull()`) so OAuth-only users can exist without a password.
   - Add `appOauthAccounts` table:
     ```
     id: text('id').primaryKey()
     userId: text('user_id').notNull().references(() => appUsers.id)
     provider: text('provider').notNull()  // 'google' | 'github'
     providerUserId: text('provider_user_id').notNull()
     createdAt: timestamp('created_at', { withTimezone: true }).notNull().defaultNow()
     ```
     With unique constraint on (provider, providerUserId) and index on userId.
   - Add `appOrgInvitations` table:
     ```
     id: text('id').primaryKey()
     orgId: text('org_id').notNull().references(() => appOrganizations.id)
     email: text('email').notNull()
     role: text('role').notNull()  // 'admin' | 'member'
     token: text('token').notNull().unique()
     invitedBy: text('invited_by').notNull().references(() => appUsers.id)
     expiresAt: timestamp('expires_at', { withTimezone: true }).notNull()
     acceptedAt: timestamp('accepted_at', { withTimezone: true })
     createdAt: timestamp('created_at', { withTimezone: true }).notNull().defaultNow()
     ```
     With index on token and composite index on (orgId, email).
   - Add to `appApiKeys`: `rpmLimit: integer('rpm_limit')` and `tpmLimit: integer('tpm_limit')` (both nullable).
   - Add to `appOrganizations`: `defaultRpmLimit: integer('default_rpm_limit')` and `defaultTpmLimit: integer('default_tpm_limit')` (both nullable).

  <action>
1. Create `src/routes/auth/oauth/google/+server.ts`:
   - GET handler: generate state + codeVerifier via Arctic, store both in httpOnly cookies (maxAge 600, path '/', sameSite 'lax'), redirect to `google.createAuthorizationURL(state, codeVerifier, ['openid', 'profile', 'email'])`.

2. Create `src/routes/auth/oauth/google/callback/+server.ts`:
   - GET handler: validate state cookie matches query param, exchange code via `google.validateAuthorizationCode(code, codeVerifier)`.
   - Decode ID token via `decodeIdToken(tokens.idToken())` to get `{ sub, email, name }`.
   - Account linking logic:
     a. Check `appOauthAccounts` for (provider='google', providerUserId=sub). If found, create session for that userId.
     b. If not found, check `appUsers` for matching email. If found, link (insert appOauthAccounts row) and create session.
     c. If no user exists, create new user with `emailVerified: true`, `passwordHash: null`, insert appOauthAccounts, create session.
   - Set session cookie using `SESSION_COOKIE_NAME`, delete OAuth state cookies, redirect to '/'.
   - On any error, redirect to `/auth/login?error=oauth_failed`.

3. Create `src/routes/auth/oauth/github/+server.ts`:
   - GET handler: generate state via Arctic (GitHub does NOT use PKCE/codeVerifier), store state in httpOnly cookie, redirect to `github.createAuthorizationURL(state, ['user:email'])`.

4. Create `src/routes/auth/oauth/github/callback/+server.ts`:
   - GET handler: validate state, exchange code via `github.validateAuthorizationCode(code)`.
   - Fetch user info from `https://api.github.com/user` with access token header.
   - If user's email is null (private email), fetch from `https://api.github.com/user/emails` and pick the primary verified email.
   - Same account linking logic as Google callback (steps a/b/c above).
   - Same session creation, cookie cleanup, and redirect.

5. Create `src/lib/components/auth/OAuthButtons.svelte`:
   - Accept `googleEnabled` and `githubEnabled` boolean props.
   - Render "or" divider: `<div class="relative my-6"><div class="absolute inset-0 flex items-center"><div class="w-full border-t border-zinc-800"></div></div><div class="relative flex justify-center text-sm"><span class="bg-zinc-950 px-2 text-zinc-400">or</span></div></div>`
   - Render full-width OAuth buttons with provider SVG logos (20px). Classes: `bg-zinc-900 border border-zinc-800 hover:bg-zinc-800 text-zinc-100 rounded-lg py-3 px-4 w-full flex items-center justify-center gap-3`.
   - Google button text: "Continue with Google". GitHub button text: "Continue with GitHub".
   - Buttons are `<a>` tags linking to `/auth/oauth/google` and `/auth/oauth/github`.
   - Only render each button if its `*Enabled` prop is true. If neither enabled, render nothing.
   - Add `aria-label="Sign in with Google"` / `aria-label="Sign in with GitHub"`.

6. Modify `src/routes/auth/login/+page.server.ts`:

7. Modify `src/routes/auth/login/+page.svelte`:
