# Phase 4 — UI Design Contract

> Visual and interaction contract for OAuth login, member management, admin API key views, per-key rate limits, and admin dashboard KPIs. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

| Property | Value |
|----------|-------|
| Tool | none |
| Preset | not applicable |

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding |
| sm | 8px | Compact element spacing, table cell padding vertical, badge padding |
| md | 16px | Default element spacing, card padding, table cell padding horizontal |
| lg | 24px | Section padding, card internal sections, panel internal padding |
| xl | 32px | Layout gaps, page section breaks |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level spacing |

Exceptions: Invite slide-out panel uses 24px internal padding. OAuth button group uses 12px gap (3 * xs) between buttons. Login page form uses 24px gap between sections (email/password vs OAuth).

## Interaction Patterns

### OAuth Login Flow
- User clicks "Continue with Google" or "Continue with GitHub"
- Browser redirects to OAuth provider's consent screen
- After consent, redirected back to `/auth/oauth/{provider}/callback`
- On success: redirected to dashboard (or org page if user has an org)
- On failure: redirected to `/auth/login?error=oauth_failed` with error message displayed
- Error display: `bg-red-900/20 border border-red-800/30 rounded-lg p-4 text-red-400 text-sm`
- If OAuth env vars not set, buttons are not rendered (no broken buttons)

### Member Invitation Flow
1. Admin clicks "Invite Member" button on Members page
2. InvitePanel slides in from right
3. Admin enters email and selects role
4. Clicks "Send Invitation"
5. Form submits via SvelteKit form action with `use:enhance`
6. On success: panel closes, pending invitation row appears in table
7. Invitee receives email with accept link
8. Clicking link: if account exists, joins org; if not, goes to signup with org pre-linked
9. After acceptance: pending row replaced with active member row

### Role Change Confirmation
- Admin clicks "Change Role" from member action menu
- Dropdown appears with role options: Member, Admin
- Selecting a role shows confirmation: "Change {name}'s role to {role}?"
- Confirm/Cancel buttons inline
- Owner role transfer: separate flow — "Transfer ownership to {name}? You will become an Admin."
- Role change submitted via form action, table row updates on success

### Member Removal Confirmation
- Admin clicks "Remove Member" from action menu
- Confirmation dialog (inline or modal): "Remove {name} from {org}? Their API keys will be deactivated."
- Confirm button: `bg-red-500 hover:bg-red-500/90 text-white`
- Cancel button: `bg-zinc-800 hover:bg-zinc-800 text-zinc-300`
- On confirm: member row removed, their keys deactivated

### Admin Key Revocation
- Admin switches to "All Keys" tab
- Clicks three-dot menu on another member's key
- Selects "Revoke"
- Confirmation: "Revoke key '{name}'? This key will immediately stop working."
- On confirm: key row shows "Revoked" status, grayed out

### Rate Limit Edit (Admin)
- Admin clicks "Edit Rate Limits" from key action menu
- Inline edit or small popover with RPM/TPM fields
- Shows current values (or "Org default: X" if inheriting)
- "Save" applies changes via form action
- Changes take effect immediately for the in-memory rate limiter

### Invite Revocation
- Admin clicks ✕ on pending invitation row
- No confirmation needed (low-risk action, can re-invite)
- Row removed from table with no animation

### Loading States
- Members table: 3 skeleton rows with pulsing cell placeholders (`animate-pulse`)
- KPI cards: skeleton with pulsing value placeholder
- OAuth buttons: show spinner icon replacing provider logo while redirect is loading

### Error States
- Failed invitation send: inline error below email field in panel
- Failed role change: toast notification with error message
- Failed member removal: toast notification with error message
- OAuth error: query param `?error=oauth_failed` renders error banner on login page

### MemberActionMenu — Interaction Specification

- **Open state:** Dropdown appears below the trigger button, positioned `absolute right-0` (right-aligned to trigger).
- **Keyboard:** Arrow keys (`ArrowUp`/`ArrowDown`) navigate menu items, `Enter`/`Space` selects the focused item, `Escape` closes the menu.
- **Focus:** First menu item receives focus on open. When the menu closes, focus returns to the trigger button.
- **Accessibility:** Trigger button gets `aria-expanded="true"` when open, `"false"` when closed. The dropdown container gets `role="menu"`. Each menu item gets `role="menuitem"` and `tabindex="-1"` (focus managed programmatically).
