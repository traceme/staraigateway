  <action>
1. Create `src/lib/server/gateway/rate-limit.ts`:
   - Implement sliding window rate limiter using a Map keyed by API key ID.
   - Interface `RateLimitResult`: `{ allowed: boolean, rpmLimit: number | null, rpmRemaining: number, tpmLimit: number | null, tpmRemaining: number, resetMs: number }`.
   - `checkRateLimit(keyId: string, rpmLimit: number | null, tpmLimit: number | null): RateLimitResult` -- evicts entries older than 60s, counts current RPM and TPM, returns whether request is allowed.
   - `recordRequest(keyId: string, tokenCount: number): void` -- records a request with timestamp and token count in the sliding window.
   - `cleanup(): void` -- evicts all stale entries from the Map, deletes empty keys. Called on a 60-second setInterval.
   - Start the cleanup interval on module load: `setInterval(cleanup, 60_000)`.
   - Export `rateLimitResponse(result: RateLimitResult): Response` -- returns HTTP 429 with JSON body `{error: {message: 'Rate limit exceeded', type: 'rate_limit_exceeded', code: 'rate_limit_exceeded'}}` and headers: `Retry-After`, `x-ratelimit-limit-requests`, `x-ratelimit-remaining-requests`, `x-ratelimit-reset-requests`, `x-ratelimit-limit-tokens`, `x-ratelimit-remaining-tokens`, `x-ratelimit-reset-tokens`.
   - Export `addRateLimitHeaders(response: Response, result: RateLimitResult): Response` -- clones response with added x-ratelimit-* headers. Only adds headers when limits are set (not null).

2. Modify `src/lib/server/gateway/auth.ts`:
   - Extend the auth lookup query to also select `rpmLimit` and `tpmLimit` from `appApiKeys`.
   - Also join or query `appOrganizations` to get `defaultRpmLimit` and `defaultTpmLimit`.
   - In the returned auth result, include `effectiveRpmLimit` (key's rpmLimit ?? org's defaultRpmLimit) and `effectiveTpmLimit` (key's tpmLimit ?? org's defaultTpmLimit). These may be null if neither key nor org has limits set.

3. Modify `src/lib/server/gateway/proxy.ts`:
   - Import `checkRateLimit`, `recordRequest`, `rateLimitResponse`, `addRateLimitHeaders` from rate-limit.ts.
   - BEFORE forwarding to LiteLLM: call `checkRateLimit(keyData.keyId, keyData.effectiveRpmLimit, keyData.effectiveTpmLimit)`. If `!result.allowed`, return `rateLimitResponse(result)`.
   - AFTER receiving response: call `recordRequest(keyData.keyId, totalTokens)` where totalTokens comes from the existing usage extraction logic. Add rate limit headers to the response via `addRateLimitHeaders(response, result)`.
   - For streaming responses: record tokens after stream completes (in the existing post-stream usage recording logic). Add rate limit headers to the initial SSE response.
