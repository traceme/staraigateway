# Phase 12 — UI Design Contract

> Visual and interaction contract for frontend phases. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none |
| Preset | not applicable |
| Component library | none (hand-rolled Tailwind components) |
| Icon library | inline SVG (hand-authored, no icon package) |
| Font | system-ui stack: system-ui, -apple-system, BlinkMacSystemFont, 'Segoe UI', Roboto, sans-serif |

**Note:** This phase adds internationalization infrastructure and a language switcher. It does not introduce new page layouts or components beyond `LanguageSwitcher.svelte`. All existing visual patterns remain unchanged. The design contract below codifies the existing codebase conventions so the executor maintains visual consistency when replacing hardcoded strings with `$t()` calls.

## Copywriting Contract

| Element | Copy (English) | Copy (Chinese) |
|---------|---------------|----------------|
| Primary CTA | "Switch Language" (label for switcher) | "切换语言" |
| Language switcher — English option | "EN" | "EN" |
| Language switcher — Chinese option | "中文" | "中文" |
| Empty state heading | n/a (no new empty states in this phase) | n/a |
| Empty state body | n/a | n/a |
| Error: invalid language | "Invalid language selection" | "无效的语言选项" |
| Error: not authenticated | "Not authenticated" | "未登录" |
| Destructive confirmation | n/a (no new destructive actions in this phase) | n/a |

**Phase-specific copywriting rules:**
1. The language switcher displays short labels ("EN" / "中文"), not full language names, to fit in the TopBar without crowding.
2. All existing English strings are extracted to `en.json` and translated to `zh.json`. The translation key structure follows the pattern defined in 12-RESEARCH.md (namespace by feature: `common.*`, `nav.*`, `dashboard.*`, `api_keys.*`, etc.).
3. Chinese translations must use standard enterprise software terminology (e.g., "成员" for Members, "设置" for Settings, "使用量" for Usage).
4. Button labels follow the pattern: verb + noun in English ("Create API Key"), verb + noun in Chinese ("创建 API 密钥").

## Language Switcher Component Spec

**Location:** TopBar, right side, between org switcher area and user menu.

**Appearance:**
- Globe icon (16x16 inline SVG, matching existing icon style: `stroke="currentColor" stroke-width="2"`) followed by current language label
- Text color: `text-zinc-300` (matches user name in TopBar)
- Hover: `hover:bg-zinc-800` (matches existing TopBar button hover)
- Container: `rounded-md px-2 py-1 text-sm` (matches user menu button)
- Dropdown: `rounded-md border border-zinc-700 bg-zinc-800 py-1 shadow-lg` (matches existing user menu dropdown)

**Interaction:**
1. Click globe button to open dropdown
2. Dropdown shows two options: "EN" (with checkmark if active) and "中文" (with checkmark if active)
3. Click option to switch — POST to `/api/user/language`, set `$locale`, call `invalidateAll()`
4. Dropdown closes after selection
5. Click outside dropdown to dismiss (same pattern as user menu)
6. No page reload — all visible text updates reactively

**Responsive:**
- Globe icon always visible (no hidden-on-mobile like user name)
- Label text ("EN" / "中文") always visible — short enough for mobile
